Apache Felix Project Info

There are different roles with which Felix community members may be associated, these roles are: users, contributors, committers, and Project Management Committee (PMC) members. These roles are assigned and assumed based on merit. Everyone in the Felix community can participate to whatever level they desire, but participating and the resulting merit gained is directly linked to the role an individual may obtain. To further understand how the Felix community operates, refer to the Community Roles and Processes document and/or join the mailing lists.

Source Code¶

This project uses git to manage its source code. Instructions on git can be found at git docs.

Web Access¶

The following is a link to the online source repository.

https://github.com/apache/felix-dev

Committer and anonymous access¶

The Felix source code can be checked out (anonymously) with the following command:

$ git clone https://github.com/apache/felix-dev

Mailing lists¶

The following mailing lists have been established for Apache Felix. For each list, there is a subscribe, unsubscribe, and archive link.

Name Subscribe Unsubscribe Archive
Felix Users - For people using Felix subprojects and/or developing bundles. Subscribe Unsubscribe www.mail-archive.com
Felix Dev - For people interested in the internal development of Felix subprojects. Subscribe Unsubscribe www.mail-archive.com
Felix Commits - For people intested in Felix subproject code changes. Subscribe Unsubscribe www.mail-archive.com

Issue Tracking¶

This project uses JIRA, a J2EE-based, issue tracking and project management application. We use it for bugs, tasks and code contributions.

Before creating a new issue, please make sure nobody has already added the issue you're reporting. When you do add an issue, please try to make it as informative as possible.

The issue tracker can be found at https://issues.apache.org/jira/browse/FELIX

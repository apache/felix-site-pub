# Describing your iPOJO components *This section describes the different features supported by iPOJO. This page aims to answer to the following question: "What can I write in my iPOJO component descriptor?"* {div:class=toc} [TOC] {div} ## Core features Core features are provided with the iPOJO runtime bundles. You can use it directly, as soon as the iPOJO runtime is deployed. * [How to require a service](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/service-requirement-handler.html) * [How to publish and provide a service](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/providing-osgi-services.html) * [How to react to lifecycle state changes](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/lifecycle-callback-handler.html) * [How to description component configuration](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/configuration-handler.html) * [How to enable/disable instance introspection](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/architecture-handler.html) * [How to act on the instance state from the implementation](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/controller-lifecycle-handler.html) ## Extensions Extensions *extend* the iPOJO model to add a specific functionality. This is made thanks to the external handler mechanism. So before using one of these features, deploy the attached external handler. * [How to receive and send events with the Event Admin](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/event-admin-handlers.html) * [How to reconfigure instances with JMX](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/ipojo-jmx-handler.html) * [How to implement an extender pattern](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/extender-pattern-handler.html) * [How to implement a whiteboard pattern](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/white-board-pattern-handler.html) * [How to wait for services and inject proxies](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-userguide/describing-components/temporal-service-dependency.html) ## A missing functionality ? iPOJO component model is extensible. So, you can implement your own handler managing you specific requirement. Refer the handler development guide ([How to write your own handler](/documentation/subprojects/apache-felix-ipojo/apache-felix-ipojo-devguide/how-to-write-your-own-handler.html)) for more details. {include:apache-felix-ipojo-footer}

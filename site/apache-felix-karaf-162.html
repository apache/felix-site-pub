Apache Felix Karaf 1.2.0 was released on June 16th 2010

This release includes these fixes/improvements:

Bug

  • FELIX-2101 - Exception in grep command
  • FELIX-2310 - Karaf framework environment is not set correctly
  • FELIX-2349 - JMX Notifications
  • FELIX-2358 - Hot undeployment of features does not uninstall all bundles
  • FELIX-2360 - Error message from bin/karaf script
  • FELIX-2361 - Unable to connect to admin:create'd instances from within the Karaf shell
  • FELIX-2371 - features-maven-plugin cannot handle classifiers
  • FELIX-2373 - Wrapper service is not configured properly in Karaf 1.6.0
  • FELIX-2381 - The default log exception renderer does not print inner exceptions anymore
  • FELIX-2382 - Installing a feature with a fragment should refresh the host
  • FELIX-2389 - Problems with security extensions in Karaf 1.6.0
  • FELIX-2395 - The client jar Class-Path manifest entry does not work anymore

Improvement

  • FELIX-2079 - Add missing metatype files to document the configurations
  • FELIX-2352 - The web console should delegate to JAAS when deployed in Karaf
  • FELIX-2364 - The width and height of the terminal should be available on the command session
  • FELIX-2365 - The help command printer should line wrap the text according to the terminal width
  • FELIX-2377 - info - Should output the exact JRE version used
  • FELIX-2404 - Use felix utils instead of various classes from karaf

Task

  • FELIX-2341 - [gogo] the commands module is not used by gogo itself and should be moved elsewhere

Known issues

  • On Ubuntu server, the SSH connection need either Bouncycastle installed as a security provider as explained in http://felix.apache.org/site/65-deploying-security-providers.html (or downloading and installing the "Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files 6" from Sun) (see FELIX-2361). This issue also happen when using the OS native service integration.

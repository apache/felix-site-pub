Licensing and other issues

  • Karaf subproject promoted to TLP.
  • Need to have discussion on how to handle provisional OSGi APIs in our releases. It is not clear if we should include provisional OSGi API since it is not officially released by the OSGi Alliance and is therefore subject to change and/or not always officially sanctioned (i.e., made publicly available by the OSGi Alliance). At a minimum, we need to define an approach to informing users if provisional API is included that it is not [yet] official OSGi API.
